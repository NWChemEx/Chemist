Detailed Description

template<typename DerivedType, typename... Particles>
class chemist::qm_operator::detail_::OperatorImpl< DerivedType, Particles >

Operator code factorization requiring particle types.

The OperatorBase class provides code factorization and interfaces for when the particles described by the operator do not need to be known. There is additional code factorization that is possible when you know the types (for example determining if something is a one or two electron operator). That's the responsibility of this class.

Note
While we refer to each type in Particles as "a particle" the reality is it can actually be an indeterminate number of particles (e.g., ManyElectrons) or a non-particle object (e.g., Density<Electron>). We opted for "particle" because it's mostly right in most cases and "object" seemed too generic.
Template Parameters
DerivedTypethe type *this is implementing. Needed for CRTP.
ParticlesA list of the interacting particles the operator describes.

template<typename DerivedType , typename... Particles>
static constexpr auto chemist::qm_operator::detail_::OperatorImpl< DerivedType, Particles >::n_electrons ( )
inlinestaticconstexpr

The determinable number of electrons in *this.

Some operators, like the one-electron kinetic energy operator, or the two-electron electron-electron repulsion operator involve a fixed number of electrons. This function determines the fixed number of electrons in *this. Note that an operator like Kinetic<ManyElectrons> is considered a one-electron operator in this sense because it contains one determinable electron as well as an indeterminable (at compile time) number of electrons.

Returns
The number of determinable electrons in *this.
