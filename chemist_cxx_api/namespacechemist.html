Detailed Description

The primary namespace for the chemist library.

All functionality of the chemist library intended for end user consumption is defined within this namespace.

◆ args_to_buffer()

template<typename T >
auto chemist::args_to_buffer ( pybind11::args  args)

Convenience function for converting pybind11::args into a std::vector.

When creating Python bindings for C++ functions which take a variadic number of inputs, the Python arguments will come into C++ as a pybind11::args object. This function wraps the process of unwrapping the pybind11::args object into a buffer, assuming that each object has the same C++ type.

Template Parameters
TThe C++ type each member of args is expected to be. The user must specify T because the compiler can not deduce it.
Parameters
[in]argsThe pack of variadic python arguments to unpack.
Returns
A std::vector<T> containing the unwrapped arguments.
Exceptions
std::bad_allocif there's a problem allocating the buffer. Strong throw guarantee.

template<typename T , typename U >
bool chemist::can_cast ( U  pybind_object)

Convenience function for checking if a pybind11 object can be cast to type T.

Template Parameters
TThe C++ type we are trying to convert the pybind11 object to.
UThe type of the pybind11 object holding the Python object. Assumed to be a type like pybind11::object

As far as we are aware, pybind11 does not provide a mechanism for determining if a cast will pass without trying the cast. This function wraps the process of trying the cast and reports whether or not the cast passed.

Parameters
[in]pybind_objectThe wrapped Python object we are trying to convert.
Returns
True if pybind_object can be converted to an object of type T and false otherwise.
Exceptions
NoneNo throw guarantee.

Adds a string representation of p to os.

This function is used to print a point to a string. The resulting format will be of the form:

x : <x-value>,
y : <y-value>,
z : <z-value>

Coordinates will be printed with all significant figures.

Warning
